# HR-Copilot 软件架构文档

## 1. 文档目的

本文档用于指导后续代码修改：先根据功能定位模块，再修改对应文件，最后运行验证并把变更记录回本文档，避免每次都从全局代码开始查找。

## 2. 根目录约定

根目录只保留入口文件、Git/IDE 元信息、运行数据目录和几个一级功能目录。

```text
HR助手/
├── main.py                  # 应用入口：HTTP 路由、接口分发、启动服务
├── README.md
├── requirements.txt
├── app/                     # 后端功能模块和前端页面
├── data/                    # 所有运行数据：简历、候选人、配置、导出、历史状态
├── docs/                    # 架构文档、维护文档
├── project_files/           # 打包、部署、说明文档、构建产物等非核心代码文件
├── .git/
├── .gitignore
└── .idea/、.agents/          # 本地工具元信息
```

`data/` 不再拆到其他目录；历史状态表已放到 `data/legacy_status/状态.xlsx`。

## 3. 模块职责

| 模块路径 | 职责 | 修改场景 |
| --- | --- | --- |
| `main.py` | HTTP API 路由、请求解析、统一调用各功能模块 | 新增接口、调整接口路径、服务启动逻辑 |
| `app/core/constants.py` | 全局路径、文件位置、流程阶段、默认提示词、默认规则 | 新增默认提示词、流程阶段、数据文件位置 |
| `app/core/storage.py` | JSON 读写、事件日志、时间工具、目录初始化 | 数据读写通用逻辑、事件日志格式 |
| `app/frontend/loader.py` | 读取前端入口页面，兼容源码运行和 exe 打包 | 前端资源加载、打包资源路径问题 |
| `app/frontend/index.html` | 前端 UI、CSS、页面交互、列表渲染、按钮事件 | 页面样式、前端交互、导航、搜索、展开收起 |
| `app/auth/service.py` | 用户、登录、会话、账号资料、密码修改 | HR/管理员账号、权限、登录态 |
| `app/config/service.py` | 岗位标准、系统配置、模型配置、提示词配置、流程规则 | 模型设置、提示词管理、岗位标准、评分阈值 |
| `app/resume/parser.py` | 简历文本解析、本地兜底字段抽取、关键词抽取 | 简历解析准确率、姓名/电话/邮箱识别、技能识别 |
| `app/collection/service.py` | 简历采集、本地上传、平台/API 采集、扫描同步 | BOSS/猎聘接口、本地文件上传、扫描数量异常 |
| `app/data/service.py` | 候选人/考试数据读写、历史状态导入、CSV 导出 | 候选人数据、培训数据、导出字段、状态表兼容 |
| `app/workflow/service.py` | 流程自动化、招聘漏斗、指标统计、重复检测、人才池过滤 | 状态自动推进、漏斗统计、提醒、去重、数据权限 |
| `app/llm/client.py` | 大模型 API 调用、模型连通测试、接口地址规范化 | LLM 连接、模型测试、API Key、超时策略 |
| `app/scoring/service.py` | 候选人规则评分、AI 复评、优势/风险/追问生成 | HR 评分规则、简历匹配分、自定义评分维度 |
| `app/jd/service.py` | JD 生成、JD 结果转岗位标准、JD 文本格式化 | JD 生成功能、岗位描述结构、岗位标准同步 |
| `app/agent/service.py` | 招聘智能体：简历结构化、综合评审、下一步推荐、群面评分、面试方案、日报 | 智能面试、群面评分、AI 评审、日报、下一步动作 |
| `app/training/service.py` | 试卷生成、考试结果保存、考试阅卷、本地评分兜底 | 智能培训、试卷生成、考试管理、阅卷规则 |

## 5. 修改定位规则

1. 页面显示异常、按钮无反应、搜索/滚动问题：先看 `app/frontend/index.html`。
2. 某个接口报错或接口路径变化：先看 `main.py`，再看被调用的 `app/**/service.py`。
3. 简历扫描数量、上传、平台采集异常：先看 `app/collection/service.py` 和 `app/resume/parser.py`。
4. 候选人列表、人才池、导出、状态同步异常：先看 `app/data/service.py` 和 `app/workflow/service.py`。
5. 状态自动推进不符合预期：先看 `app/workflow/service.py` 和 `app/config/service.py`。
6. 提示词、模型、评分阈值、岗位标准配置：先看 `app/config/service.py` 和 `app/core/constants.py`。
7. AI 返回质量、超时、模型连接失败：先看 `app/llm/client.py`，再看具体业务模块。
8. 面试方案、群面评分、AI 综合评审、招聘日报：先看 `app/agent/service.py`。
9. JD 生成和岗位标准自动生成：先看 `app/jd/service.py`。
10. 智能培训、试卷和阅卷：先看 `app/training/service.py`。

## 6. 验证流程

每次修改后至少执行：

```powershell
python -m py_compile main.py app\core\constants.py app\core\storage.py app\frontend\loader.py app\auth\service.py app\config\service.py app\data\service.py app\resume\parser.py app\collection\service.py app\workflow\service.py app\llm\client.py app\jd\service.py app\scoring\service.py app\training\service.py app\agent\service.py project_files\packaging\exe_launcher.py
```

建议再执行轻量导入检查：

```powershell
python -c "import main; print('import ok'); print(len(main.load_index_html(main.ROOT)))"
```

涉及页面时，重启 11011 服务后访问：

```text
http://127.0.0.1:11011
```

如果浏览器仍显示旧页面，使用 `Ctrl + F5` 强制刷新。

## 7. 打包注意事项

前端页面位于 `app/frontend/index.html`，exe 打包必须携带 `app/frontend` 目录。

当前打包相关文件：

- `project_files/packaging/build-exe.ps1`：从项目根目录执行 PyInstaller，输出到 `project_files/build_artifacts/dist`。
- `project_files/packaging/HR-Copilot.spec`：使用项目根目录作为 `pathex`，并携带 `app/frontend`。
- `project_files/packaging/exe_launcher.py`：exe 双击启动器。
- `app/frontend/loader.py`：兼容源码目录和 PyInstaller `_MEIPASS` 临时目录。
