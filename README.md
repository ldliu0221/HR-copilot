# HR Copilot

HR Copilot 是一个面向招聘流程的本地化智能体应用，围绕简历采集、简历解析、岗位标准、候选人评分、流程跟进、面试方案、培训考试和招聘数据分析进行设计。

## 核心功能

- 数据看板：展示候选人总数、推进中人数、平均匹配分、推荐推进率、招聘漏斗、智能提醒和 AI 招聘日报。
- 简历采集：支持本地简历导入、平台候选人文本导入，以及按个人账号配置招聘平台采集接口。
- 候选人库：集中管理候选人信息、简历解析结果、岗位匹配分、来源、阶段和更新时间。
- 简历评分：支持按岗位标准进行评分，输出优势、风险、追问建议和推进建议。
- 自定义评分标准：可维护岗位画像、技能要求、能力要求、风险关键词、通过分和评分规则。
- 流程自动化：支持手动修改候选人阶段，也支持根据群面评价、面试反馈、作业提交等节点自动同步状态。
- JD 生成：根据职位名称、部门、工作地点、薪资范围、岗位职责、任职要求和技能要求生成结构化 JD。
- 智能面试：根据不同候选人的简历和岗位要求生成个性化面试方案、考察重点和追问问题。
- 智能培训：包含试卷生成、考试管理、考试结果查看和导出。
- 提示词管理：集中维护简历解析、简历评审、JD 生成、面试方案、考试评分、日报生成等功能使用的提示词。
- 系统设置：支持模型接口、API Key、招聘平台采集接口、个人账号配置和管理员权限设置。

## 产品设计重点

本项目按照 HR 使用场景优化智能体能力：

- JD 生成：通过行业知识库和 AI 生成降低 JD 模板化、岗位细节不清晰的问题。
- 简历抓取：通过标准化接口和个人账号配置解决招聘平台数据格式不统一的问题。
- 简历评分：通过多维度评分规则和 AI 复核降低主观评分偏差。
- 考试评分：通过标准答案模板、关键词场景和人工复核结合提升评分稳定性。

## 技术说明

- 后端：Python 标准库 HTTP Server
- 前端：原生 HTML、CSS、JavaScript
- 数据存储：本地 JSON、JSONL、CSV 文件
- 文档解析：PDF、DOCX、TXT、Markdown
- 表格处理：openpyxl
- 可选模型接口：兼容 OpenAI 风格 Chat Completions API

## 本地开发运行

```bash
pip install -r requirements.txt
python main.py
```

默认访问地址：

```text
http://127.0.0.1:11011
```

## 目录说明

```text
main.py              应用主程序，包含页面、接口、数据处理和智能体逻辑
requirements.txt    Python 依赖
README.md           项目说明
```

运行时会在本地生成 `data/` 目录，用于保存候选人、配置、状态、导出文件和简历解析结果。该目录包含业务数据，不应提交到代码仓库。

## 注意事项

- 本仓库只保留功能相关代码文件。
- 不提交 exe、打包产物、启动脚本、Docker 脚本、服务器检查脚本、本地简历数据和缓存文件。
- 平台接口、模型 API Key 和个人账号信息应仅保存在本地配置中，不应写入仓库。
