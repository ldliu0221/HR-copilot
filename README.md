# HR Copilot

> 面向招聘流程的 **HR 智能体工作台**，覆盖 JD 生成、简历采集、简历解析、候选人评分、流程自动化、智能面试、培训考试与招聘数据分析。

---

## 项目简介

HR Copilot 是一个面向 HR 招聘场景的本地化智能体应用。系统围绕「信息沉淀、流程跟进、数据分析」三个核心目标设计，帮助 HR 将分散在简历文件、招聘平台、面试反馈、群面评价、考试结果中的信息统一沉淀到候选人库，并通过规则评分与大模型能力辅助完成候选人筛选、推进建议、面试方案生成和招聘复盘。

项目当前采用轻量化架构实现，便于在本地快速运行、演示和二次开发。未配置大模型时，系统会使用本地规则兜底；配置兼容 OpenAI Chat Completions 的模型接口后，可启用 AI 解析、AI 评审、JD 生成、面试方案、考试评分和日报复盘等能力。

---

## 核心能力

- 简历采集：支持本地文件导入、平台候选人文本导入，并预留 BOSS、猎聘等招聘平台的个人账号接口配置。
- 简历解析：支持 PDF、DOCX、TXT、Markdown 等格式，提取姓名、联系方式、岗位、学历、技能、项目经历和工作经历。
- 候选人库：统一管理候选人基础信息、简历来源、当前阶段、匹配分、更新时间和 AI 分析结果。
- JD 生成：根据职位名称、部门、工作地点、薪资范围、岗位职责、任职要求和技能要求生成结构化 JD。
- 岗位标准：支持维护岗位画像、技能要求、能力要求、风险关键词、通过分和评分规则。
- 简历评分：按照岗位标准输出匹配分、优势、风险、追问建议和推进建议。
- 流程自动化：支持手动修改候选人状态，也支持根据群面评价、作业提交、面试反馈等节点自动同步状态。
- 智能面试：根据候选人简历和岗位要求生成个性化面试方案、考察重点、风险追问和评价维度。
- 智能培训：包含试卷生成、考试管理、答题结果查看、得分统计和结果导出。
- 提示词管理：可维护简历解析、简历评审、JD 生成、面试方案、考试评分、日报生成等功能提示词。
- 数据看板：展示候选人总数、推进中人数、平均匹配分、推荐推进率、招聘漏斗、智能提醒和 AI 招聘日报。
- 权限与配置：支持普通 HR、管理员角色，以及模型接口、平台采集接口和个人账号配置。

## 项目结构

```text
.
├── main.py              # 应用主程序，包含页面、接口、数据处理和智能体逻辑
├── requirements.txt     # Python 依赖
├── README.md            # 项目说明
└── .gitignore           # 忽略运行数据、打包产物和本地配置
```

运行后系统会自动生成本地数据目录：

```text
data/
├── resumes/             # 简历文件入口
├── status/              # 候选人、事件、状态同步数据
├── config/              # 岗位标准、提示词、模型和平台配置
└── exports/             # CSV 导出结果
```

`data/` 目录包含业务数据、账号配置和候选人信息，不应提交到代码仓库。

## 使用流程

### 1. 录入或采集简历

将简历文件放入本地数据目录，或在「简历采集」页面上传 PDF、DOCX、TXT、Markdown 文件。系统会解析候选人基础信息，并写入候选人库。

```text
简历文件
  -> 简历解析
  -> 候选人结构化信息
  -> 候选人库
```

### 2. 配置岗位标准

在「岗位标准」中维护岗位画像、技能要求、能力要求、风险关键词和通过分。评分规则会影响批量评分、AI 评审和推进建议。

```json
{
  "name": "AI 算法实习生",
  "required_skills": ["Python", "RAG", "Prompt Engineering"],
  "abilities": ["工程实现", "问题拆解", "项目交付"],
  "risk_keywords": ["频繁跳槽", "项目描述空泛"],
  "pass_score": 70
}
```

### 3. 批量评分候选人

点击「批量评分」后，系统会按岗位标准计算匹配分，并输出优势、风险和追问建议。启用模型接口后，可进一步使用 AI 复核评分。

```text
候选人简历
  -> 岗位标准
  -> 本地规则评分
  -> AI 综合评审
  -> 推进建议
```

### 4. 跟进招聘流程

候选人支持以下阶段流转：

```text
待沟通 -> 已沟通 -> 待群面 -> 已群面 -> 待提交群面作业
-> 已提交群面 -> 待最终面 -> 已最终面 -> 淘汰 / 通过
```

HR 可以手动修改阶段，也可以让系统根据群面评价、作业提交、面试反馈等节点自动同步状态。

### 5. 生成面试方案

在「智能面试」中选择候选人后，系统会结合简历、岗位标准和评分结果生成面试方案，包括技术问题、项目追问、风险核验和评价标准。

### 6. 管理培训考试

在「智能培训」中可生成试卷、维护考试、查看考生得分与得分率，并导出考试结果。

## 示例接口

### 扫描简历

```text
POST /api/scan
```

示例请求：

```json
{
  "remove_missing": true
}
```

示例返回：

```json
{
  "scanned": 12,
  "total": 105,
  "removed_count": 3
}
```

### 批量评分

```text
POST /api/score-all
```

示例请求：

```json
{
  "profile_id": "ai-intern"
}
```

示例返回：

```json
{
  "updated": 105,
  "average_score": 76.3,
  "recommended_rate": 0.667
}
```

### 导出候选人

```text
GET /api/export/candidates.csv
```

返回候选人 CSV 文件，可用于表格归档或同步到外部文档。

## 模型与提示词

系统默认使用本地规则完成基础解析和评分。管理员可在系统设置中配置兼容 OpenAI Chat Completions 的模型接口：

```json
{
  "llm_enabled": true,
  "llm_base_url": "https://api.openai.com/v1/chat/completions",
  "llm_model": "gpt-4o-mini",
  "llm_api_key_env": "OPENAI_API_KEY"
}
```

提示词管理覆盖以下任务：

- 简历结构化解析
- 简历综合评审
- JD 生成
- 面试方案生成
- 群面评价与作业评分
- 考试题目生成
- 考试结果评分
- 招聘日报生成
- 候选人下一步动作建议

## 技术亮点

- 轻量化单文件应用：核心逻辑集中在 `main.py`，便于演示、迁移和二次开发。
- 本地数据优先：业务数据默认落在本地 `data/` 目录，适合低成本试点和内网部署。
- 规则评分 + AI 复核：未接入模型时可用规则兜底，接入模型后增强解释性和灵活性。
- 招聘流程自动化：将候选人阶段、面试反馈、群面作业和推进建议串联成闭环。
- 提示词可配置：关键 AI 功能的提示词可在页面中维护，便于 HR 根据岗位和业务风格调整。
- 平台采集可扩展：支持按 HR 个人账号配置招聘平台接口，适配不同来源的数据采集。

## 后续优化方向

- 增加更严格的候选人去重策略，按手机号、邮箱、姓名和简历指纹合并重复记录。
- 增加更细的评分维度权重，例如技能、项目、稳定性、学历、沟通表达和岗位意愿。
- 增加面试官协同评价，支持多人打分、评价汇总和冲突提醒。
- 增加招聘平台真实 API 适配器，完善 BOSS、猎聘、智联等来源的数据同步。
- 增加操作审计与权限隔离，区分个人数据、团队数据和管理员数据。
- 增加测试覆盖，重点覆盖简历解析、状态同步、评分规则和导出逻辑。

## 项目总结

HR Copilot 将招聘工作中高频、重复、分散的信息处理流程整合为一个本地智能体工作台。它既可以作为 HR 招聘流程自动化的 MVP，也可以作为后续接入企业系统、招聘平台 API 和大模型能力的原型基础。
